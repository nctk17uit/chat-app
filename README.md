# Chat App

A Flutter chat application powered by Firebase. Users can sign in or register with email/password, pick a profile image, and send messages. Messages appear in a Messenger‑style UI and are stored in Cloud Firestore. A Cloud Function publishes push notifications for new messages.

## Features

- Email/password authentication (Firebase Auth)
- Profile image upload (Firebase Storage)
- User list stored in Firestore
- Real-time chat with message bubbles and avatars
- Push notifications via Firebase Cloud Messaging and a Firestore-triggered Cloud Function
- Desktop, mobile and web support (Android, iOS, macOS, Windows)

## Setup

1. **Clone the repository**
   ```bash
   git clone https://github.com/<your-username>/chat_app.git
   cd chat_app
   ```

2. **Install Flutter dependencies**
   ```bash
   flutter pub get
   ```

3. **Firebase configuration**
   - The project already includes generated `lib/firebase_options.dart` and
     platform config files (`android/app/google-services.json`, etc.) from
     the FlutterFire CLI. If you need to connect to your own Firebase project,
     run `flutterfire configure` and overwrite these files.
   - Ensure the Firebase project has Authentication, Firestore, Storage and
     Cloud Messaging enabled.

4. **Cloud Functions (optional)**
   ```bash
   cd functions
   npm install
   firebase deploy --only functions
   ```
   The included function listens for new Firestore documents under `chat/`
   and sends FCM notifications.

5. **Running the app**
   ```bash
   flutter run        # selects a connected device or simulator
   flutter run -d chrome  # run on web
   ```

6. **Environment variables / secrets**
   - API keys in `firebase_options.dart` are public identifiers; no extra
     env configuration is required unless you prefer to externalize them.
   - You may use `--dart-define` or packages like `flutter_dotenv` if you
     wish to keep some settings out of source control.

## Project structure

- `lib/` – main app code
  - `screens/` – authentication and chat screens
  - `widgets/` – message bubble, image picker, etc.
  - `firebase_options.dart` – Firebase initialization data
- `functions/` – Cloud Functions source
- `android/`, `ios/`, `web/`, etc. – platform folders generated by Flutter
